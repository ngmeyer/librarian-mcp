# Research Frameworks

Before running a topic through the 6 lenses, choose the framework that best fits the question type. This shapes how each lens approaches its analysis.

## Framework Selection

### Causal Analysis
**When:** "Why is X happening?"
**Approach:** Trace cause-effect chains. Each lens identifies different causal factors.
**Output emphasis:** Root causes ranked by evidence strength.

### Comparative Analysis
**When:** "How does X compare to Y?"
**Approach:** Structured comparison across dimensions. Each lens provides different comparison criteria.
**Output emphasis:** Decision matrix with trade-offs.

### Trend Analysis
**When:** "Where is X heading?"
**Approach:** Identify trajectories and inflection points. Each lens projects from its angle.
**Output emphasis:** Scenario mapping (base, bull, bear cases).

### Systems Analysis
**When:** "How does X work?" or "Why does this system behave this way?"
**Approach:** Map actors, feedback loops, incentives. Each lens illuminates different system dynamics.
**Output emphasis:** System map with leverage points.

### Risk Assessment
**When:** "What could go wrong with X?" or "Should we do X?"
**Approach:** Enumerate risks from each lens perspective. Weight by probability and impact.
**Output emphasis:** Risk matrix with mitigation options.

### Stakeholder Analysis
**When:** "Who is affected by X and how?"
**Approach:** Map stakeholders and their interests. Each lens reveals different stakeholder motivations.
**Output emphasis:** Stakeholder map with alignment/conflict zones.

## How to Choose

1. Read the research question
2. Identify the question type (why, how, what-if, compare, who)
3. Select the matching framework above
4. If multiple fit, default to **Systems Analysis** — it's the most general
5. Note your framework choice in the output header

## Connection to Lenses

The framework shapes the *questions* each lens asks, not the lens itself. For example:
- [[technical]] under Causal Analysis asks "what data shows the cause?"
- [[technical]] under Trend Analysis asks "what does the data trajectory show?"
